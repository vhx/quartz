Media Object

The media object is so useful you won't believe it. Life with the media object is drastically different than it ever was before. Note that padding and alignment is added between elements, using our utility classes as necessary.

All children elements of a media object (element with class "media") should have the "media-unit" class. Think of each of these units as table cells. The media unit you want to fill the remaining area should have the class "media-fill". The other media units will automatically be contained to the size of their content. In cases where you have text content in a media unit that you wish to not wrap, simply add the "media-nowrap" class. See examples below.

Javascript Plugin API

In order to make use of the percentage button you first must get the plugin API as shown below.


// get the element
var api = $('[data-button-percent]').data('button-percent-api');

Once you have the api, there are several javascript API methods available.

Method Description
start():void Start the percentage progress. Button text will change to '0%'.
change(percent:int):int Set the percentage of the button. Button text and bar will change to reflect the provided percentage
reset():void Reset the button back to its original state.

The above methods also trigger events on the DOM element, whicy you can listen for if/as necessary.

Event Description
button-percent:start Triggerd on button percent start (see above event).
button-percent:change Triggerd on button percent change (see above event). 

$('[data-button-percent]').on('button-percent:change',function(event, data){
  // data will be an integer (0-100) representing the percentage state
});

$('[data-button-percent]').on('button-percent:start',function(event){
  // does not return any data
});

    Javascript Plugin API

    In order to make use of the percentage button you first must get the plugin API as shown below.

    
// get the element
var api = $('[data-range]').data('range-api');

    Once you have the api, there are several Javascript API methods available.

    Method Description
    setValue(value:int):void Set the value of the slider. The slider will update/round the value provided based on the increment and if the relative_to_min option is set.
    setColor(hex:string) Set the color of the slider. You can set the color with CSS or use this method to set the color after the factor (say when the slider hits a certain range).

    The above methods also trigger events on the DOM element, whicy you can listen for if/as necessary.

    Event Description
    range:ready Triggered when the range slider has been initiated and all events bound. An alternative method to retrieving the plugin api.
    range:change Triggered on slider value change and returns the value and percentage of the range.
    range:min Triggered when the slider value hits the min value of the range. Returns the value and percentage.
    range:max Triggered when the slider value hits the max value of the range. Returns the value and percentage. 
    
$('[data-range]').on('range:ready',function(event, data){
  // data will be the instance of the range plugin (and api)
  var api = data;
});
$('[data-range]').on('range:change',function(event, data){
  // data.percent.simple will be an integer (0-100) representing the percentage (rounded)
  // data.percent.real will be a float of the actual percent basis
  // data.value will be an integer/current value of the slider range
});
$('[data-range]').on('range:min',function(event, data){
  // data.percent.simple will be an integer (0-100) representing the percentage (rounded)
  // data.percent.real will be a float of the actual percent basis
  // data.value will be an integer/current value of the slider range
});
$('[data-range]').on('range:max',function(event, data){
  // data.percent.simple will be an integer (0-100) representing the percentage (rounded)
  // data.percent.real will be a float of the actual percent basis
  // data.value will be an integer/current value of the slider range
});

    Javascript Folder Structure

    Setup

    Ignite

    Setup

    Ignite files allow for a single place to organize what specific Javascript will be included and initialized on the page. It is a custom approach (as opposed to using something like require.js) as this allows us the customization and flexibility we need to integrate with our class/plugin system and the rails pipeline.

    A global ignite file exists per primary section (admin, home, etc) to intialize any global scripts. Each page/section then has its own ignite file which is included on the page with its own correlating folder of js classes/modules. It essentially is for defining 3 things.

    1. We can specify all vhx and vendor plugins we need included (via the rails pipeline) at the top.
    2. We then can specify all the elements we would like to be able to reference withing our classes.
    3. We then list all plugins (vhx and vendor) as well as classes that need to be initialized (either on ready or on load).

    Here's an example of an ignite file:

    
// --- vhx plugins ---
//= require vhx.plugin.plugin-name.js

// --- vendor plugins ---
//= require highcharts.js

// --- directory ---
//= require_directory ./dashboard

var VHX = VHX || {};
VHX.Section = VHX.Section || {};

/*.....................................
  Activity Dashboard Section
......................................*/
VHX.Section['Dashboard'] = function() {
  var self = this;

  /* elements
  ....................................*/
  self.elements = {
  }

  /* init classes
  ....................................*/
  self.classes = {
    onready: { },
    onload: { }
  };

  /* init plugins
  ....................................*/
  self.plugins = {
    onready: { },
    onload: { }
  };

  /* vendor plugins
  ....................................*/
  self.vendor = {
    onready: { },
    onload: { }
  };

  self.instance = new VHX.Ignite(self);
};

    Plugins

    Setup

    When new javascript functionality for a given page element is required, consider first if the said functionality could be useful elsewhere. If so, writing a jquery plugin is recommended.

    Note: Plugins are always tied to a specific element. If the functionality is not tied to a specific element (where abstraction is more confusing than its worth) then see Writing Classes.

    Plugin Structure

    First we need to create our plugin and add it to the app/assets/javascripts/_shared/plugins folder, using a naming convention of vhx.plugin.plugin-name.js. Using the provided structure (below) will automate much of the process of initalizing and creating the plugin instance.

    
/*.....................................
  VHX Plugin
......................................*/
VHX.Plugin['plugin-name'] = VHX.Super.extend({

  /* default options
  ..................................*/
  options: { },
  elements: { },

  /* setup
  ..................................*/
  setup: function() {
    // setup will automatically be fired
    var self = this;

    self.customMethod();
    self.bindEvents();
  },

  /* ready event
  ..................................*/
  ready: function() {
    var self = this;

    // this.elements.el is the element you attached the plugin to
    // use events to allow for hooks into the plugin
    self.elements.el.trigger('plugin-name:ready');
  },

  /* bind events
  ..................................*/
  bindEvents: function() {
    var self = this;

    self.ready();
  },

  /* custom method
  ..................................*/
  customMethod: function() {
    var self = this;

    self.elements.el.trigger('plugin-name:custom');
  }
});

    You can add whatever additonal methods you need to the object. Remember that this represents the plugin object itself. Keep in mind however, that the scope of this will change in certain cases (say inside a setTimeout as shown above). Where necessary, you can create a reference to this by using var self = this; at the top of your method.

    Initializing the plugin

    For each element you would like your plugin to be initialized on, you can add a data attribute with the plugin's name.

     <!-- Element where plugin is attached -->
<div data-plugin-name></div>

    If you have custom options for that particular instance you can simply define them within the data attribute like so:

     <!-- Element where plugin is attached -->
<div data-plugin-name="option: value; option: value;"></div>

    Finally, you'll need to add a reference to the plugin in the section ignite file (See more about what these are). You can specifiy whether you want the plugin to initialize on page ready or page load, as well as adding any global options for plugin instances on the page. 

    
/* init plugins
....................................*/
self.plugins = {
  onready: {
    'plugin-name': { option: 'value', option: 'value' }
  },
  onload: {

  }
};
    Plugin APIs

    Sometimes you may want to call methods of a plugin instance via javascript. In using the above method of setting up your plugin, you can easily access all the methods of the plugin via its api. To retrieve it, simply do the following:

    
/* plugin api
..................................*/

var api = $('[data-plugin-name]').data('plugin-name-api');

api.pluginMethod();
    Plugin Events

    You may need to execute certain code on events triggered by plugins. For instance, to ensure you retrieve the plugin api, it would be safer to wait until you know for sure the plugin is ready. You could instead do the following:

    
/* plugin events
..................................*/
var api;

$('[data-plugin-name]').on('plugin-name:ready',function() {
  api = $('[data-plugin-name]').data('plugin-name-api');
});

api.pluginMethod();

    In writing our own plugins we can specifiy as many events and where events should be fired, allowing us greater control and flexiblity.

    Plugin: Toggle
    • Filename: vhx.plugin.toggle.js
    • Description: provides click-to-toggle functionality (see: /admin Site Switcher, /settings Password Change)
    • Markup: 
       <!-- /* toggle plugin markup
..................................*/
<div class=“{toggle-name}” data-toggle=“width_of_parent: false; close_on_html_click: true”>
  <button data-toggle-trigger=”{toggle-name}”></button>
  <div data-toggle-target=“{toggle-name}”></div>
</div> -->
    • Options (set inline, as value of data-toggle):
      • width_of_parent - true sets data-toggle-target to width of data-toggle || false allows custom CSS-based width setting for data-toggle-target
      • close_on_html_click - true to close data-toggle-target when clicking outside it || false close data-toggle-target only when data-toggle-trigger is clicked.
    Plugin: Flash

    See Github PR for documentation for the time being, until we move things to Quartz.

    Classes

    Setup

    When page specific javascript functionality is required and is not necessarily tied to a specific element, it should be placed in one or more Javascript classes/modules. The goal is to break up functionality into modules to avoid a mishmash of random javascript methods. By forcing ourselves to organize each page's javascript into separate files we can make iteration and management of page javascript easier.

    We can use the same basic setup as we did with our plugins with a few minor differences.

    

var VHX = VHX || {};
VHX.Class = VHX.Class || {};

/*.....................................
  VHX Class
......................................*/
VHX.Class['class-name'] = VHX.Super.extend({

  /* default options
  ..................................*/
  options: { },
  elements: { },

  /* setup
  ..................................*/
  setup: function() {
  // setup will automatically be fired
    var self = this;

    self.bindEvents();
  },

  /* ready event
  ..................................*/
  ready: function() {
    var self = this;

    // use events to allow for hooks into the class
    self.elements.el.trigger('class-name:ready');
  },

  /* bind events
  ..................................*/
  bindEvents: function() {
    var self = this;

    self.elements.el.on('click', function(e) {

    });

    self.ready();
  },

  /* custom method
  ..................................*/
  customMethod: function(e, self) {
    var self = this;

    self.elements.el.trigger('class-name:custom');
  }
});

    You can add whatever additonal methods you need to the object. As with plugins, remember that this represents the plugin object itself. Keep in mind however, that the scope of this will change in certain cases (say inside a setTimeout as shown above). Where necessary, you can create a reference to this by using var self = this; at the top of your method.

    Initializing the Class

    You'll just need to add a reference to the class in the section ignite file (See more about what these are). You can specifiy whether you want the class to initialize on page ready or page load, as well as adding any options for class instances on the page. 

    
/* init classes
....................................*/
self.classes = {
  onready: {
    'class-name': { option: 'value', option: 'value' }
  },
  onload: {

  }
};
    Referencing Elements

    If you have elements that you'll need access to in the class, you can either specify them within the class itself (under the elements property) or at the top of the section ignite file (shown below). The advantage of doing it in the ignite file, is that these elements will be available to you for all classes that have been initialized on the page. This will help avoid multiple references to the same element.

    
/* elements
....................................*/
self.elements = {
  container: $('.container'),
  area: {
    container: $('.area'),
    block: $('.block')
  }
};

// within the class elements will be available like so
this.elements.container
this.elements.area.block
